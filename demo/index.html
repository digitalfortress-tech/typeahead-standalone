1. Autocomplete with Local Data (Identifier + Groups)

Try typing the letter "G" or "P"...

The code shown below can be edited in place and its changes reflected back in the input box above).
  • If you tried typing the letter "G" or "P", you should have seen some suggestions pop up instantly
  • USING IDENTIFIER: Lets use another data source => source: colors2. As soon as you update the source, you will see an error - Missing Identifier. This is because colors2 is an array of objects. For an array of objects, the Identifier property is mandatory as it represents the text that should be displayed as the suggestions. Therefore, lets set the identifier to identifier: 'name'. When you try searching for "B", you should see some suggestions pop up immediately.
  • USING GROUPS: Lets say you wish to group colors by their shade. We will use the source as source: colors2. Since colors2 is an array of objects, we will set the identifier: 'name' as explained above. Now to group colors by their shade, we need to set the groupIdentifier property which represents the name of the group to be used. So, we set groupIdentifier: 'shade'. Once again, when you try searching for "B", you should see some colors grouped by their shade. Also, if you notice, the color "Brown" has no group since it has no "shade" property.
var colors1 = ['Green', 'Grey', 'Gold', 'Pink', 'Purple'] // You can use *group* to group colors too var colors2 = [ { name: 'Red Deep', shade: 'Shades of Red' }, { name: 'Blue Dark', shade: 'Shades of Blue' }, { name: 'Blue Light', shade: 'Shades of Blue' }, { name: 'Brown' }, { name: 'Black', shade: 'Shades of Black' }, { name: 'Black Light', shade: 'Shades of Black' }, ]; typeahead({ input: document.getElementById('liveSearch1'), highlight: true, source: { local: colors1, // try using "colors2" // identifier: 'name', // groupIdentifier: 'shade', } });

Note: With typeahead, when you search for multiple words, the order of the words does not matter. To explain this easily with an example, use colors2 as the source, and then try searching for "light" or "light blue".

2. Autocomplete with Remote Data (retrieved via AJAX)


  • Try searching for "Fr" and you should see some search results loaded from the remote endpoint. If you had kept the network tab of your developer tools open, you would have seen the necessary requests made.
  • Network Requests Cached: Now, if you clear the input and type "Fr" again, you will see the search results again but this time no requests are made. This is because, in typeahead, the results of every XHR request are cached to avoid loading the same data again.
  • Network Requests on demand: In the previous step, when you searched for "Fr", 2 requests were made and their results were added to the search index. If you now type "Ma", you will see countries beginning with the letters "Ma" without any network requests being made. A network request will be made on demand only when the number of available suggestions is lesser than the maximum limit of suggestions which by default is 5. Try typing "Mal" and you should see a new network request made since the search index returned less than 5 matches.
typeahead({ input: document.getElementById("liveSearch2"), highlight: true, source: { remote: { url: 'https://restcountries.com/v2/name/%QUERY', wildcard: '%QUERY', }, identifier: 'name', }, });

3. Autocomplete with Prefetched Data (retrieved via AJAX)


  • To understand the working of this example, be sure to keep the network tab of your developer tools open. As soon as you focus on the search box above, a network request will the made to prefetch data. This happens because we set when: 'onFocus' which causes data to be prefetched on focussing the search input element. By default, this property is when: 'onInit' meaning that data is prefetched / loaded as soon as typeahead gets initialized. The done flag is optional & can be used to disable the prefetch request programmatically. Its default value is false. It gets set to true automatically when data is prefetched for the first time (to prevent multiple network requests). By setting done: true, the prefetch request will not occur. The process(suggestions) callback is optional. It gets executed after the prefetch request occurs. It receives the transformed suggestions as a parameter & as an example can be used to store the received suggestions in localStorage to be used later.
  • Now try typing "A"... you will see suggestions pop up instantaneously without making any request since the data was already fetched & added to the search index.
  • Prefetch can be used in conjunction with remote to fetch more precise results.
typeahead({ input: document.getElementById("liveSearch3"), highlight: true, source: { prefetch: { url: 'https://raw.githubusercontent.com/digitalfortress-tech/typeahead-standalone/master/demo/assets/json/countries.json', when: 'onFocus', // default is `onInit` done: false, // optional, default => false process: (items) => {}, // optional, default => undefined }, identifier: 'name', }, });

4. Typeahead-standalone with Templates

The following JS fiddle demonstrates how you could style every aspect of your typeahead instance by using the config option - templates.
  • Using templates is completely optional. There are 5 types of templates and you can use a few of them or all of them according to your needs. All the templates can optionally contain HTML too which is parsed and rendered as expected. All the templates accept a callback and they MUST return a string.
  • Header, Footer: The header and footer templates are callbacks that each return a string(text/HTML) and are rendered at the top and the bottom of the suggestions list respectively. In the result tab below, when you search for "Red", you should see the header ("Colors Found") at the top of the list and the footer ("See more") at the bottom of the list.
  • Suggestion: If you searched for "Red", you would see the suggestion template. The suggestion template is a function that receives the current suggestion/item as a parameter. This paramater can be used to display not only the suggestion's text, but also additional data like colors as seen in the example. You can add any valid HTML which will be rendered within the suggestion allowing you to fine tune your styling.
  • Group: The group template is a function that receives the name of the group as a parameter. In the result tab below, when you start typing "Bl", you should see several suggestions which are arranged by their group (groupIdentifier). This template allows you to modify the styling of the group's element.
  • Not Found: The notFound template is a callback that returns a string (text/HTML) and is rendered when no suggestions are available. In the example, if you search for a non-existent color like "xyz", you should see the notFound template in action.



5. Autocomplete with Data Tokens and custom display option


  • DATATOKENS: Datatokens are used to add additional data to the search index. To understand this easily, in the example below, see the songs data source. We are using identifier: 'title' which means songs will be searchable by their title. But what if we wanted to also search by artist and by album ? This is where dataTokens comes in. We simply set dataTokens: ['artist', 'album'] which causes these properties to be added to the search index.
  • Try to search by artist/album, (for example, try "Christmas" or "Paul") and you should have the expected suggestions displayed. If you search for "good", notice how the position of the word "good"" in the title does not matter while being searched.
  • CUSTOM "display()": If you try searching for "it", you should see a single suggestion pop up. Try to select it (either by clicking on the suggestion or by pressing the "Tab" key). You will see that by default the title of the track (i.e. the identifier) gets displayed as the value of the search box. What if you wanted to include the artist name as well ? To do so, uncomment the display function below and retry. The display callback receives the selected item as its parameter. You can use it to generate a string and return that string which will be used. as the input's value. By default, the identifier's text is used but you can customise the default behaviour in this callback
var songs = [ { title: 'Emmanuel', artist: 'Blackmores Night', album: 'Christmas Songs', }, { title: 'Henei Ma Tov (Behold how good)', artist: 'Paul Wilbur', album: 'Shalom Jerusalem', }, { title: 'Angels We Have Heard On High', artist: 'Fernando Ortega', album: 'Christmas Songs', }, { title: 'It is good', artist: 'Paul Wilbur', album: 'Jerusalem Arise', }, { title: 'The Shout Of El Shaddai', artist: 'Paul Wilbur', // album: 'Jerusalem Arise', } ]; typeahead({ input: document.getElementById("liveSearch5"), highlight: true, source: { local: songs, identifier: 'title', dataTokens: ['artist', 'album'], }, // display: (selectedItem) => { // return selectedItem.title + ' - ' + selectedItem.artist; // }, templates: { suggestion: (item) => { return item.title + ' - ' + item.artist; // return item.title + ' (' + item.album + ')'; }, }, });

Typeahead-standalone Comparison

This library is inspired by the original Twitter-typeahead library with a few performance improvements and a few additional features.
Let's see the differences between typeahead-standalone and other popular libraries...

1. Twitter-typeahead

  • Depends on jQuery
  • Does not support grouped suggestions
  • No support for prefetching suggestions
  • Supports fewer templates than typeahead-standalone
  • It is 10 times larger in terms of size (~40kb minzipped vs ~4kb minzipped for typeahead-standalone)
  • As of 2022, not maintained since 8 years

2. Autocomplete

  • Does not have a true suggestion engine
  • Does not support hints
  • Does not support identifiers/tokenizers (search based on multiple tokens)
  • Does not support prefetching of remote data
  • Does not support templating
  • Unsupported data transformation for remote endpoint

3. Suggestions

  • depends on Fuzzy and xtend
  • Does not support fetching remote data
  • Does not have a true suggestion engine i.e. Fuzzy matching takes place
  • Does not support hints
  • Does not support style templates
  • has very few customization options
